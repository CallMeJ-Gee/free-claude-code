Getting Started

1. Install Requirements

Install Claude Code and the required dependencies:

curl -LsSf https://astral.sh/uv/install.sh | sh
uv self update
uv python install 3.14

2. Clone and Configure

Clone the repository and configure your environment:

git clone https://github.com/Alishahryar1/free-claude-code.git
cd free-claude-code
cp .env.example .env

3. Configure a Provider

Edit .env and add your provider credentials:

NVIDIA_NIM_API_KEY="nvapi-your-key"
MODEL="nvidia_nim/z-ai/glm4.7"
ANTHROPIC_AUTH_TOKEN="freecc"

4. Start the Proxy

Start the proxy server:

uv run uvicorn server:app --host 0.0.0.0 --port 8082

5. Run Claude Code

Run Claude Code with the proxy:

ANTHROPIC_AUTH_TOKEN="freecc" ANTHROPIC_BASE_URL="http://localhost:8082" claude

Providers

NVIDIA NIM

High-quality hosted models with excellent performance.

  • Get API key at build.nvidia.com/settings/api-keys
  • Popular models: nvidia_nim/z-ai/glm4.7, nvidia_nim/z-ai/glm5
  • Best for: Production use, high-quality responses

OpenRouter

Access to many models including free options.

  • Get API key at openrouter.ai/keys
  • Free models: open_router/stepfun/step-3.5-flash:free
  • Best for: Testing, personal projects, cost savings

DeepSeek

Affordable high-quality models with Anthropic-compatible API.

  • Get API key at platform.deepseek.com/api_keys
  • Popular models: deepseek/deepseek-chat, deepseek/deepseek-coder
  • Best for: Coding tasks, affordable production use

Local Providers (LM Studio, llama.cpp, Ollama)

Run models locally for privacy and offline use.

  • No API key required
  • Requires local hardware with sufficient resources
  • Best for: Privacy, offline use, cost savings

Configuration

Model Routing

Configure different models for different tiers:

MODEL="nvidia_nim/z-ai/glm4.7"           # Default model
MODEL_OPUS="nvidia_nim/z-ai/glm5"      # Opus tier
MODEL_SONNET="open_router/..."        # Sonnet tier
MODEL_HAIKU="lmstudio/..."            # Haiku tier

Rate Limiting

Control request rate to avoid hitting provider limits:

PROVIDER_RATE_LIMIT=1              # Requests per window
PROVIDER_RATE_WINDOW=3             # Window in seconds
PROVIDER_MAX_CONCURRENCY=5         # Max concurrent requests

Timeouts

Configure timeout values for provider requests:

HTTP_READ_TIMEOUT=120             # Read timeout
HTTP_WRITE_TIMEOUT=10             # Write timeout
HTTP_CONNECT_TIMEOUT=10           # Connect timeout

Troubleshooting

Claude Code shows "undefined ... input_tokens"

This usually indicates an issue with streaming response format. Check:

  • ANTHROPIC_BASE_URL is http://localhost:8082 (not /v1)
  • The proxy is returning Server-Sent Events for /v1/messages
  • server.log contains no upstream 400/500 response before the error

Provider returns HTTP 400

This usually means the local runtime rejected the request. Check:

  • The local server supports POST /v1/messages
  • The model supports the requested context length and tools
  • llama.cpp was started with enough --ctx-size
  • The configured base URL includes /v1 for LM Studio and llama.cpp

Provider disconnects during streaming

Errors like "incomplete chunked read" usually come from the upstream. Try:

  • Reduce concurrency
  • Increase timeouts
  • Retry later

FAQ

How do I use the model picker in Claude Code?
Set ANTHROPIC_BASE_URL to point to the proxy, then run /model in Claude Code. The proxy will list all available models from configured providers.
Can I use multiple providers at the same time?
Yes! Configure multiple providers in your .env file and use MODEL_OPUS, MODEL_SONNET, and MODEL_HAIKU to route different tiers to different providers.
Is this proxy secure?
The proxy supports authentication via ANTHROPIC_AUTH_TOKEN. Set this to a secure value and only expose the proxy on trusted networks.
How do I run this in production?
Use a process manager like systemd, Docker, or supervisord to manage the proxy. Ensure you set appropriate rate limits and timeouts for your use case.

  • GET /admin/routing - Model routing configuration
  • GET /admin/settings - Settings management
  • GET /admin/logs - Logs viewing
  • GET /admin/system - System status
  • GET /admin/health/ - Health check
  • GET /admin/metrics/overview - Metrics overview

Proxy API Endpoints

  • POST /v1/messages - Create message (streaming)
  • POST /v1/messages/count_tokens - Count tokens
  • GET /v1/models - List available models
